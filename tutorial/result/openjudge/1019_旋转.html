1019:旋转

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
10000ms
单个测试点时间限制:
1000ms
内存限制:
256000kB
描述
在消除游戏当中,红色和蓝色的方块进入了一个N*N的区域中.整个区域在竖直平面上,这样由于重力作用所有的方块都会垂直下落到该列的最下方的空白区域.例如,看下面两个例子:

在这些图片中,每一个’.’表示一个空白区域,每一个’R’表示该区域被一个红色方块占据了,每一个’B’表示该区域被一个蓝色方块占据了.其中左边的例子是正确的,但右边的是错误的.这是因为在第三列的一个方块(用箭头指出)并没有掉落到它下面的那一格.
如果一个玩家能让相同的颜色在同一行，或者同一列，或者同一条斜线上连续有k个存在的话他就赢了,下面列出了四种可能:



这样一来,你就在进行这个消除游戏了,并且你有一次作弊的机会来保证你胜利!当你的对手走神的时候,你可以将你的平台区域顺时针旋转90度.由于重力的作用,方块都会掉落然后变成下面这个样子:



不幸的是,在对手注意到这一切之前你只有旋转它一次的机会.
剩下的问题就是找好一个合适的时间来进行旋转.给出一个游戏盘面,你需要算出哪个(或是哪些)玩家可以在顺时针旋转之后在重力作用下掉落后凑成k个方块.
输入
输入数据的第一行表示了测试数据的数目T.在接下来的T组数据中,每一个数据的开头都包含了两个整数N和K.接下来的N行每一行都有N的字符串长度,表示了游戏平台的区域,使用了和上述描述一样的格式.
在给出的位置中,每一组数据都是”正确的位置”.特别的,不会有哪个玩家一开始就形成了k个连续需方块.
输出
对于每一组测试数据,输出一行,为"Red", "Blue", "Neither",或是 "Both"中的一个,意思是这个或这些玩家会有k个连续的该颜色的方块可以消除(Neithe表示两种颜色都没有连续的K个, Both表示两种颜色都有连续的K个)

Neither

提示
你只可以旋转一次
请确保重力只在你完全旋转之后才生效
请确保只有重力生效后才去检查赢家

3 ≤ N ≤ 50.
1 ≤ T ≤ 100.
3 ≤ K ≤ N.
