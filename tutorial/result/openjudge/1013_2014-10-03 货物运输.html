1013:2014-10-03 货物运输

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
20000ms
单个测试点时间限制:
2000ms
内存限制:
262144kB
描述

Mr.Lin 最近在做运货的工作。他每日运货需要经过的城市有N 个，它们依次排列在一条双
向高速公路上。我们可以将这条高速公路看作数轴，将第i 个城市看做数轴上的点xi，并且
满足x_{i + 1} > x_i。
Mr.Lin 所在单位位于城市1。Mr.Lin 需要从单位运货到城市N 后再从城市N 返回单位。由
于路程很长，Mr.Lin 可能有必要在途中停下加油。具体来说，在Mr.Lin 开车到达一个城市
后，他可以选择直接继续前行，也可以选择停下车来加油后继续前行。然而，加油是需要掏
银子的。在城市i 加油需要花ci 两银子。注意这里加油的费用与加油的量无关，无论加多少
油，费用都是相同的。有的城市的加油站相当黑，这样Mr.Lin 就会被剥削得很惨。好在Mr.Lin
所在的单位比较良心，在单位内部加油不需要银子，即c1 = 0。另外，如果途中加油产生的
费用不超过L 两银子，Mr.Lin 也可以找单位报账。但是Mr.Lin 知道，油是要用油箱来装的，
是油箱就得有个容量，是货车其任意时刻的油就不能超出油箱的容量，而油箱越大越贵。
Mr.Lin 希望你能为他计算出在保证能完成任务且不在加油这件事上自己掏银子的前提下所
需油箱的最小容量。
另外，已知体积为V 的油恰好可以支持货车前行V 的距离。Mr.Lin 初始时油箱为空。

输入
第一行两个正整数N,L，分别为城市的数量及报账的上限。
第二行N 个整数xi 表示城市的位置。
第三行N 个整数ci 表示加油的花费。
输出
第一行输出一个整数表示最小油箱的容量V。
第二行输出2N - 1 个数依次描述Mr.Lin 每次到达一个城市时的加油方案。其中第i 个数描
述第i 次到达一个城市时的加油情况（除城市N 仅出现一次外每个城市均出现两次）。具体
地，每行输出一个整数A（0 <= A <= V，表示此次在到达的城市中加的油量为A。如果A =
0，表示此次没有加油。若方案有多种，任意输出一种即可。
样例输入
5 1
1 2 3 4 5
0 2 3 4 5
样例输出
8
8 0 0 0 0 0 0 0 0
提示
数据约束
对于20%的数据，N <= 100。
对于40%的数据，N <= 1000。
对于70%的数据，N <= 50000。
对于100%的数据，2 <= N <= 500000，|xi| <= 10^9，c1 = 0，0 <= ci <= 10^9，1 <= L <= 10^12。
大约有50%的数据满足xi = i，且分布大致均匀。

由于openjudge不支持spj，这里只输出第一行就行了。要检验第二行是否正确请自己用lemon评测。
