7:沙漠国王

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
3000ms
内存限制:
65536kB
描述

David大帝已经成为沙漠王国的国王，为赢取民心，他决定建立一些供水渠道，使每个村庄都有水渠能用来浇灌田地。水可以从首都通过渠道送往任何相连的村庄。作为国家的统治者和这个国家智慧的相征，他需要用一种非常优雅的修建方案。

经过多天的学习之后，他终于制定出了一个计划。他想让每公里渠道的平均花费尽可能小，换种说法就是：总花费与总长度的比率尽可能小。他想要的只是让水连向所有的村庄，也就是说任何两个村庄之间只可能有一种连通方案。

工程师通过调查，记录了所有村庄的高度与位置。所有的水渠必须完全水平，并且不能弯曲。由于任意两个村庄都在不同的高度，所以他们最终得到结论是他们必须使用一种使水流向上或向下垂直输送的设备来改变水流的高度。渠道的长度取决于两个村庄的水平距离，而渠道的花费取决于垂直输送设备送水时需要输送水的高度。你应该注意的是不会有两个村庄有相同的高度，而且任何两个渠道不能共享同一个设备。渠道可以相交而不对对方产生影响，并且不会有任何三个村庄在一条直线上。作为David大帝的首席科学家与首席程序员，你决定找到一种最优的建造方案。

输入

有多组测试数据，每组测试数据第一行都是一个整数N(2 <= N <= 1000),表示村庄的个数。随后的N行每行包含三个整数x,y,z（0 <= x,y < 10000,0 <= z < 10000000).(x,y)表示村庄的位置，z表示村庄的高度，第一个村庄表示首都。当输入的N等于0时表示输入结束，它不需要当作一组数据来进行处理。

输出

对于每组测试数据，输出一个十进制小数，用以表示总花费与总长度的比率，精确到小数点后三位

样例输入
4
0 0 0
0 1 1
1 1 2
1 0 3
0
样例输出
1.000
提示

给定一些结点，每个结点表示为（x,y,z），其中（x,y）表示结点在平面坐标轴的坐标，z表示高度。如果在两个结点之间连线，那么他们在平面坐标轴上的距离是长度，高度差是花销。现在请找到一种连线方式将结点之间进行连接，使得所有结点可以构成一个连通图，并且使得总花销与总长度的比例尽可能小。

输入：
第1行：一个整数n，表示结点数量
第2行：一共n行，每行代表一个结点，用三个整数分别代表x，y，z

输出：
总花销与总长度的比例，精确到小数点后三位。
例如输出m，精确到小数点后三位可以写作：printf("%.3lf\n",m)
