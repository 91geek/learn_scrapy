1011:2014-10-03 砍数

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
10000ms
单个测试点时间限制:
1000ms
内存限制:
262144kB
描述

兰可夫天生仇恨数字，因而他最喜欢干的事情就是把一堆数摆在一起，把它们砍成段段。
放国庆了，兰克夫选了一堆正整数，开始砍。当然，他砍数是有一定规则的。具体来说，他
会将每个正整数x 按照如下算法进行砍的动作：
Cut(x)
    cnt = 0
    while x <> 0
        cnt = cnt + 1
        x = x / 10
兰克夫发现，在砍一些数的过程中（即在上面的while 循环内），有的数会变成一个存在于
最初的数堆中的数（一个数本身也可算作可在砍的过程中变成自己）。兰克夫希望知道，对
于每个初始数堆中的数x，假如我们在初始的数堆中选出那些可以在砍的过程中变成x 的数，
然后在这些数中等概率随机选择一个数来砍，将其砍成x 时变量cnt 的期望值。

输入
第一行一个整数N 表示数的数量。
接下来N 行每行一个正整数xi。
输出
输出N 行。其中第i 行为xi 的答案。如果答案为整数，直接输出整数，否则，输出一个最简分数，即形如p/q，且gcd(p, q) = 1。
样例输入
5
123
1234
1
10
100
样例输出
1/2
0
8/5
1/2
0
提示
数据约束
令L 为所有数的长度之和。
对于30%的数据，L <= 5000。
对于100%的数据，1 <= N, L <= 1000000。保证每个正整数无前导零，不保证不出现重复正整数。
