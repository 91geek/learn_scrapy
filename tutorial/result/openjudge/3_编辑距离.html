3:编辑距离

  • 查看
  • 提交

总时间限制:
1000ms
内存限制:
4000kB
描述

编辑距离，又称Levenshtein距离（也叫做Edit Distance），是俄罗斯科学家Vladimir Levenshtein在1965年提出的概念。

它是指两个字串之间，由一个转成另一个所需的最少编辑操作次数。许可的编辑操作包括将一个字符替换成另一个字符、插入

一个字符和删除一个字符。

例如将kitten一字转成sitting：
     sitten  （k->s）
     sittin  （e->i）
     sitting （ ->g）

因此kitten和sitting的编辑距离是3。

（本题源自www.51nod.com）

输入
求编辑距离的次数n；后面跟2*n行，每两行为一组，一组中
第1行：字符串a(a的长度 <= 1000)。
第2行：字符串b(b的长度 <= 1000)。
输出
输出n行，每行均为a和b的编辑距离
样例输入
2
kitten
sitting
Gocd
Giood
样例输出
3
2
提示
假定字符串中不包含空格、制表符和回车换行符。
