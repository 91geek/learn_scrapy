I:做梦的小玛尼

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
1000ms
内存限制:
256000kB
描述

众所周知，小玛尼是一个爱睡觉的小朋友。这一天他梦见自己骑着马闯关。关卡地图是一个 n * m 的网格，第 i 行第 j 列的格子里会有 w[i][j] 个布丁。他可以选择第一行中任意地点出生，且每路过一个点可以获得当前格子里的所有布丁。他的马是中国象棋里面的马，假设他当前所在格子坐标为 (i, j)，那么他只能走到 (i + 1, j + 2), (i + 1, j - 2), (i + 2, j + 1), (i + 2, j - 1) 四个格子中的一个（当然，他不能走出网格）。他必须到达最后一行中的一个格子才算闯关成功。梦做到这里，他被馋醒了，于是拜托孙神帮他计算梦中的他应该如何行进才能获得最多的布丁，而他本人去 711 买布丁了。孙神觉得他已经无可救药了，于是想帮他找出一条路径使得他只能获得最少的布丁。

输入
第一行包含两个整数，分别为 n 和 m (1 ≤ n, m ≤ 1000)
接下来的 n 行每行 m 个数，其中的第 i 行第 j 列表示 w[i][j] (w[i][j] ≤ 100000)
输出
输出应包含一行，2 个整数，即最多可能获得的布丁数和最少可能获得的布丁数
样例输入
2 4
10 0 1 9
1 2 0 3
样例输出
11 2
提示
最多：从 (1, 4) 出发，走到 (2, 2) 结束
最少：从 (1, 3) 出发，走到 (2, 1) 结束

不知尼之梦为布丁与，布丁之梦为尼与？
