1001:声明转换

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
1000ms
内存限制:
65536kB
描述

弄清楚C++中的声明语法是一件令人如此头疼的事。虽然可以引入typedef简化声明，或者使用左右法则进行阅读，
但是依然需要花很长时间读懂代码，比如：



int (*(*(*func)(int *))[5])(int *)


其中func是一个函数指针，这类函数参数类型是int指针返回类型是指向数组的指针，所指向的数组的元素也是函数指针，指向的函数具有int*形参，返回类型为int。

在C++的声明中*是前缀，[]和()是后缀，于是在出现歧义的时候就需要加上括号，这是声明可读性不强的原因。

在《C++语言的设计和演化》中Bjarne曾经提到可从通过引入后缀式的-$>$替代前缀声明符*来改变这个情况，于是声明如下：


old declarationsnew declarations
int* v[10];int v:[10]->;
int(*v)[10];int v:->[10];
int (*(*(*func)(int*))[5])(int*)int func:->(int*)->[5]->(int*);


这样的话，从左往右读就能知道被声明的变量的类型了。


现在，baihacker想把前缀和后缀式的声明转换为新的后缀式声明（线性声明），你能帮助他吗？


在这里，前缀后缀混合式声明使用简化的声明语法：



simple-declaration:
	intdeclarator;
declarator:
	direct-declarator
	ptr-operatordeclarator
direct-declarator:
	identifier
	direct-declarator[constant-expression]
	direct-declarator(parameter-declaration-clause)
	(declarator)
ptr-operator
	*
	const
const-expression为非零无符号32位整数；
parameter-declaration-clause为用,隔开的simple-declaration序列或者为空；
identifier为只含字母的序列。

参考资料:

部分例子来源于：
http://blog.csdn.net/fisher_jiang/archive/2006/07/29/995441.aspx
后缀式声明参考自:
Bjarne stroustrup, The Design and Evolution of C++, China Machine Press, 2002, pp:22-23.
前缀后缀混合式声明参考:
ISO/IEC 14882:2003(E), A.6-A.7: 683-686

输入
第一行是一个整数T，表示测试数据的组数。

接下一每一行是一个测试数据。

限制：

每一行不超过1000个字符；

输入的声明总是合法的；

结果中不会有连续两个const在一起；

函数的返回类型不会是数组或函数，也没有函数构成的数组；

输入中可能会有自由出现的空格，如果不影响声明的话。
输出
对于每一个测试数据，将答案输出到一行当中。

对于函数的参数列表部分（`('和`)'间的字符）只需要原封不动地打印出来，即使有前导或者后置空格。

观察样例得到更多信息。
样例输入
7
int * v[10];
int (*v)[10];
int (*(*(*func)(int*))[5])(int*);
int (*(*func[7][8][9])(int*))[5];
int (*(*func)[1][6])[7][8];
int const*const*func[5];
int const*const*const func[5];
样例输出
int v:[10]->;
int v:->[10];
int func:->(int*)->[5]->(int*);
int func:[7][8][9]->(int*)->[5];
int func:->[1][6]->[7][8];
int func:[5]->const->const;
int func:[5]const->const->const;
