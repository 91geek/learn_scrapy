1025:Memory Usage

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
3000ms
内存限制:
65536kB
描述

Fungi's super computer has many processes running on it all the time. Each process will be allocated some consecutive memory, in the form of some leading hex numbers and an asterisk. For example, a (16×8)-bit super computer may allocate DEAD*, which is in fact the memory from DEAD0000 to DEADFFFF. A memory location can be shared by multiple processes. When a new process is created, it needs to find a best location in the address space where as few processes are running there as possible. You are here to help determine how many processes are using a given address space, or its subspace.

输入
The first line has an integer T, the number of test cases.
Each test case has three integers on the first line, k (1<=k<=128), n and m (1<=n,m<=5×10^4). The next n lines will be strings no longer than k, each indicating the address usage of one process. The string may, but not necessarily, end with an asterisk for the meaning described above. Then there will be m lines as the queries, in the same format of the address usage.
输出
For each case, output m integers, one on each line, for each query respectively.
样例输入
1
10 3 2
DEAD*
DEADB*
DEADBEEF
DEA*
DEADB*
样例输出
3
2
