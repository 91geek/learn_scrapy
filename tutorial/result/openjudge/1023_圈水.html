1023:圈水

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
3000ms
内存限制:
131072kB
描述

zyx发现了一片神奇的水域。这片水域可以简化为如下的二维模型：在平面直角坐标系中，将横向方向看做是宽度，水域可以看做是沿着横向向两侧无限延伸的；将纵向看做高度，并且这片水域的高度为h，水深视为无限，换言之，在坐标系中，y=h以下的面积均为水域部分。

到了这里，zyx突然想到了一个非常无聊的游戏——圈水，于是他往这片水域中投下了许许多多的隔板，将水域分成了好几块，然后他希望取走位于全部的隔板之上的水，严格的讲，zyx的每个隔板可以看做是一条y=ax+b的直线，当且仅当一个水内坐标(x0,y0)对于所有的隔板y=aix+bi（i为标号），均满足y0>aix0+bi时，我们认为这个坐标属于zyx选中的区域。当然，由于这样取水很容易就会导致最终zyx会取走无穷大面积的水，于是他事先规定了一个范围(l,r)，只有水域中满足l<=x<=r的水他才会取走。（假设在取水的时候水不会流动）

于是聪明的你能否帮助zyx计算一下他最终会取走的水的面积是多少呢？

输入
第一行只有一个整数T(T<=20)，表示测试数据组数。
对于每组数据，第一行四个整数n,l,r,h,其中n表示zyx的隔板个数；l,r,h的含义如题目描述所说。（n<=100000,-10000<=l<=r<=10000,|h|<=10000）
接下来n行，每行两个整数ai,bi，表示第i个隔板直线的参数。（|ai|<=100,|bi|<=10000）
输出
对于每一组数据，输出占一行，为zyx取走的水的面积，要求四舍五入到2位小数。
样例输入
2
2 0 2 1
-1 1
1 -1
2 0 2 0
-1 1
1 -1
样例输出
1.00
0.00
提示
对于第一个样例，[0,2]区间内，y>-x+1,y>x-1,y<1围城的水域面积为1。
对于第二个样例，[0,2]区间内，y>-x+1,y>x-1,y<0围城的水域面积为0。
四舍五入的输出方式为("%.2f\n",ans),其中2处为需要的精确位数。
输入数据量很大，对于C/C++的选手，不建议使用cin作为输入，建议使用scanf的方式读取数据。对于Java的选手，不建议使用Scanner，建议使用BufferedReader,实例化的方式为new BufferedReader(new InputStreamReader(System.in)).
