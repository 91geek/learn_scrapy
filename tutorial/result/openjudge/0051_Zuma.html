10000ms
单个测试点时间限制:
1000ms

256000kB
描述

Sinatra做出了一个祖玛游戏，他现在就来邀请你体验一下。很不幸的，这个游戏的得分函数是用Java2K完成的（一个所有函数均有一定几率返回随机值的语言），导致了这个游戏的得分充满了不确定性，但经过了漫长的观察，你最终还是发现了游戏的规律：

游戏开始时共有k种不同颜色小球，且共形成了n块，每块均有10^t,(0<=t<=3)颗同色小球（不同块颜色仍可能相同），且这n块排成了一排。你每次可以消去连续的同色小球，然后剩余的两段球会并为一段。在消去全部小球后游戏结束。在每次的得分判定中，你每消去连续的k颗第i种小球，将得到k^Pi的分数（1<=pi<=4）。但由于Java2K的不确定性，对于第i种球的每一颗均有Ci的概率在消去时出现BUG，变为无色。（注意无色球不得分，且会破坏颜色的连续，得分会在所有要消去的球确定颜色后再计算，之后包括五色球在内的所有球会消失）。

Sinatra承诺你的方法只要平均下来得分大于10^15就给你发大奖，你自然想知道采用最优方案时得分的期望值，现在你弄到了所有该有的数据，但Sinatra自然不会帮你算，只能靠你自己来完成了。


输入
第一行两个整数n，k，t， 含义如上文所示。
第二行n个整数Gi（1<=Gi<=k）,表示第i块小球的颜色。
第3～k+2行，每行先给出一个整数Pi，然后是一个实数Ci，含义如描述中所示。
输出
一个实数， 表示你采用最优方案时得分的期望值，保留至小数点后3位。
样例输入
5 2 0
1 2 1 2 1
4 0.5
2 0
样例输出
16.750
提示
[样例解释]
样例中最优方案为先消去第二颗小球，期望1.0，序列变为{1,1,2,1}，再消去第三颗小球，期望1.0，序列变为{1,1,1}，最后将剩下的全部消去，期望为（81+16+16+2+1+1+1+0）/8=14.75，故总期望为16.75
【数据范围】
对于30%数据保证k=1
对于另外40%的数据保证对任意Ci(1<=i<=k)存在Ci = 0。
对于100%的数据保证 1<=n<=200 , 1<=k<=max(n,15)， 0 <= Ci < 1。 数据中所有实数小数部分不超过2位
数据不保证每种小球都出现
数据保证任意小球总数量均不大于50000
保证答案不大于10^15
