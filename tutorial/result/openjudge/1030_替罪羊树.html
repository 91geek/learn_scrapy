1030:替罪羊树

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
10000ms
单个测试点时间限制:
1000ms
内存限制:
512000kB
描述

磨子桥技校的 yzjc 教授正在教百脑汇附中的 hja 同学一种很厉害的数据结构:替罪羊 树。它是一棵不用旋转的平衡树。 现在 yzjc 教授向 hja 同学展示了一棵替罪羊树,树上有 n 个果子。hja 想摘一些果子 来给 yyy 吃。 每个果子有一个高度 hi 和一个采摘用时 ti。hja 想摘下一个果子时必需从第上一个果子 的高度爬到下一个果子的高度。假设不管向上还是向下,hja 爬一个单位高度的时间为一个 单位时间。(意思是 hja 从四楼的机房到五楼的教室只用爬一层楼) 现在 yyy(不应该是 yzy 么?)想知道 hja 在 T 个时间内最多能摘到多少个果子? (我们假设 hja 可以从任意一个位置开始摘果子)


输入
第一行一个正整数 n,表示果子的数目。
第二行 n 个正整数 ti,表示采摘第 i 个果子所花费的时间。
第三行 n 个正整数 hi,表示第 i 个果子的高度。
第四行一个正整数 T,表示时间限制。
输出
输出一行一个非负整数,表示 hja 最多能摘到的果子数量。
样例输入
4
3 5 4 11
2 1 3 1
17
样例输出
3
提示
对于 20%的数据,1 ≤ n ≤ 10。
对于 40%的数据,1 ≤ n ≤ 30, 1 ≤ T ≤ 3,000。
对于 100%的数据,1 ≤ n ≤ 50, 1 ≤ T ≤ 10,000,000。
对于 100%的数据,1 ≤ hi, ti ≤ 100,000。
