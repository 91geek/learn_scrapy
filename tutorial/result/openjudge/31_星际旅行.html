31:星际旅行

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
10000ms
单个测试点时间限制:
1000ms
内存限制:
1000000kB
描述
xzl是一个星际航行家，他要去K星系，在K星系中有n颗星球编号（0---n-1）,星球之间有m条航线，当然也有可能有航线是这颗星球自己到自己，两颗星球间不止一条航线也是太正常了。每条航线花费的旅行时间不一样的。
现在，xzl从0号星球出发，目的是去编号最后一颗星球。xlz的旅行时间是有限的，为了充分利用时间，他决定让整个旅行时间恰好是T。他只想走马观花的看一下每一颗星球，所以，在每颗星球他不做任何停留。
xzl虽然非常聪明，但是不想花时间在思考旅行方案，所以问你，他是否能够刚好T小时到达n-1星。
若可以恰好到达输出“Possible”否则输出“Impossible”。(不含引号)。


输入
第一行一个正整数Case，表示数据组数。

每组数据第一行3个整数，分别为n, m, T。

接下来m行，每行3个整数x, y, z,代表星球x和星球y之间有一条耗时为z的双向边。
输出
对于每组数据输出”Possible”或者”Impossible”.
样例输入
2
3 3 11
0 2 7
0 1 6
1 2 5
2 1 10000
1 0 1
样例输出
Possible
Impossible
提示
【样例解释】
第一组：0 -> 1 -> 2 :11
第二组：显然偶数时间都是不可能的。

【数据范围】
30%: T <= 10000
另有30%: n <= 5 , m <= 10.
100%: 2 <= n <= 50 , 1 <= m <= 100 , 1 <= z <= 10000 , 1 <= T <= 10^18 , Case <= 5.
