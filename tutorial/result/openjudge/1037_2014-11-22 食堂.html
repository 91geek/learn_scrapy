1037:2014-11-22 食堂

10000ms
单个测试点时间限制:
1000ms

描述

食堂提供十分美味的饭菜，但是其拥挤程度常常足以败坏其在众食客心中的形象。
现在Mr.Lin 正在考虑中午饭在哪里吃。如果食堂不拥挤Mr.Lin 便会去食堂，否则他宁愿
去校门口吃锅盔。于是他先调查了一下食堂里的人数，以方便做出决定。
Mr.Lin 来到食堂，发现这里有N 个座位排成一行，从左至右依次标号为1; 2; ... ;N，每个
座位至多只能坐一个人。一个人一个人数效率太低了，为了提升自己的办事效率，Mr.Lin 决定
只选择M 段连续的座位，对每段分别数出其中的人数。由于食堂里十分嘈杂，Mr.Lin 无法专
心数数，因此他怀疑自己数错了。但是Mr.Lin 想当自信地认为，一定没有数漏的情况，即所有
错误都是有人被重复计数所致（Mr.Lin甚至可能在空气中看到了人影）。现在，他把自己得到的数据交给了你，希望你帮他算出，食堂
里最多有多少个人。

输入
第一行两个整数N;M，分别为座位数量和Mr.Lin 选的段数。
接下来M 行每行三个整数l; r; k，表示Mr.Lin 数了l 到r 这一段座位，给出的结果为k。
输出
输出一行一个整数表示答案。
样例输入
4 2
1 2 1
2 4 2
样例输出
3
提示
对于20% 的数据，N;M <= 20。
对于40% 的数据，N;M <= 500。
另有20% 的数据，Mr.Lin 选的每段座位不相交。
对于100% 的数据，1 <= N;M; k <= 10^5; 1 <= l <= r <= N。
