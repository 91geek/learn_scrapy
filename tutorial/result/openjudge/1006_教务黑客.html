描述

当我们登录交大教务系统时，除了要输入用户名，还要有密码，那么在教务数据库系统里，会存储我们的用户名和密码吗？

显然，肯定要，不然什么人都可以登录教务系统。但是还是存在一个问题：如果某黑客盗取了数据库里的数据，那么我们的用户名和密码岂不是都被盗了？

所以，可以想象，教务系统肯定会尽量避免这个问题，一个常用的方法是，对用户的密码进行Hash运算（可以简单理解为加密），当用户输入用户名和密码，只需要验证对应用户名的密码的Hash值即可，举例说明如下：

某同学的学号是：20150001，密码是：888888，那么在数据库系统中并不是直接存储20150001和888888。

而是：一方面直接存储20150001，另一方面对密码888888进行某种运算，得到一个映射值（即Hash值），假设对888888运算后的值是12345678，即：Hash（888888）=12345678。

然后，接着将12345678存入数据库中。这样一来，即使有黑客攻击，也不会出现密码泄露，因为没有直接存储密码。

好了，如果该同学要登录系统，该怎么办呢？方法很简单：

该同学依次输入学号20150001和密码888888，然后，当服务器端收到密码888888，它会对888888进行Hash运算，得到12345678，然后发现与数据库里面存储的Hash值相同，验证通过。

输入
输入第一行包含一个整数T，表示测试数据的组数；接下来的每一行包含一个整数H，表示密码的Hash值。数据范围：1 <= T <= 10^4。输入保证Hash值合法，有至少1个原始密码与之对应。
输出
对于每一组测试数据，输出测试序号（从1开始），以及对应的答案，如果有多个原始密码对应相同的Hash值，则按【从小到大】依次输出，相邻两个密码间用1个空格隔开即可。注意，每个密码输出占6位，如果你算出一个密码少于6位，则需要补足前导0到6位，如1对应的是000001；另外，密码001023对应整数1023，因此，在调用get_hash(pas,ind)函数时，参数pas的值就是001023对应的整数值，即1023。
样例输入
2
0
2146174820
样例输出
Case #1: 000000
Case #2: 185783 989703
提示
假设我们使用的计算机都是32位的操作系统和程序编译环境。

对于第一组测试数据，当pas取000000，ind取0、1、2、3、4、5、6、7时，可得到hash值为0，所以只需要输出000000即可；

对于第二组测试数据，当pas取185783，ind取7时，可得到hash值2146174820，或者当pas取989703时，ind取2时，也可以得到hash值2146174820，所以输出185783和989703。
