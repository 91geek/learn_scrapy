1015:HTML

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
1000ms
内存限制:
65536kB
描述

Anna wants to learn web coding. So she is must first learn the syntax of HTML. A typical simple HTML page will look like:


<html>

<body>

<h1>heading</h1>

<p>paragraph</p>

</body>

</html>


There are a lot of angular brackets <>. Each pair of brackets ... define a context, where the type of the context is given in the brackets, and the content is in between. Contexts may be embedded into other contexts. As a beginner, Anna probably would write some invalid HTML. Your task is to help her analyze whether her code is valid.


输入
The first line is the number of cases T.
Then T lines follow. Each line represent a case of the HTML written by Anna. The HTML type and content is generalized, which means there can be any string as the type name and any string as the content. However, to simplify things, all strings will consist only lower-case letters (no spaces). A single string (consecutive lower-case letters) will not contain more than 10 characters but empty strings are allowed as contents or context names. The total length of one HTML line will not exceed 10^5.
输出
Output, for each case on a separate line, whether it's "valid" or "invalid" HTML. Special characters <, >, / can only be used to define contexts. An HTML code is invalid if and only if
1) there is one pair of context definition, or the type name in it, mismatch;
2) there is one special character at a wrong place.
样例输入
2
firsthtml
helloworld


badworld
样例输出
valid
invalid
