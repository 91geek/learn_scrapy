1099:2013-05-20 nanae的新欢

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
30000ms
单个测试点时间限制:
3000ms
内存限制:
512000kB
描述

nanae和她心爱的爱酱分手了…因为适妞是其中的小三。适妞跟nanae在一起的时候，特别爱吃醋，不想让nanae提及一点儿关于爱酱的事情。

适妞要求nanae 换一个ID，因为nanae的这个ID已经跟cxlove紧紧地联系在了一起。

nanae 挑选ID是很挑剔的，她不想让适妞看见他的ID里面有一丝跟爱酱有关的词。 这些词被称作“敏感词”。

适妞也不能让nanae瞎选ID，他也希望nanae选择的单词和自己有关的。同时他希望ID的长度与原来ID的长度相同，于是适妞规定了nanae的新ID的上下区间（也就是字典序必须介于这两个字符串之间，包含两端）

nanae想问你，他现在有多少种选择能满足适妞的要求。

因为nanae 智力有限，他只能算到2013*2013。所以，你只要输出答案对于(2013*2013) 取模的值即可。

P.S: nanae的ID仅由小写字母组成。


输入
输入数据的前两行是两个长度相等的由小写字母组成的字符串，S，T。 代表nanae ID 的上下界。
下面一行是一个整数 N，代表有N个敏感词，是nanae新的ID里不能出现的。
接下来 N 行，每行一个字符串 P，代表不能出现的敏感词。
下面一行是一个整数 M，代表有M个适妞希望出现在 nanae 新 ID 里面的词，每个单词至少需要出现一次。
接下来M行每行一个字符串 Q，代表适妞想出现在nanae 新 ID 里的词。
输出
输出一个整数，代表nanae 可以选择的ID个数。
样例输入
ilovecxlove
iloveshiniu
1
cxlove
1
shiniu
样例输出
1
提示
对于所有的数据，S的字典序在T之前，它们的长度<=2013。
N<=10，M<=3
字符串P和Q的长度均不超过10
