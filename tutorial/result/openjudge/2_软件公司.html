2:软件公司

总时间限制:
10000ms
单个测试点时间限制:
1000ms
内存限制:
65536kB
描述

Elly开始为一个大型软件公司工作。这个公司的雇佣关系是一棵树。在那里每一个人（除了老板）都恰好有一个直接上司。这个公司被分成若干个团队。每一个程序员和他的直接上司和非直接上司形成一个团队。这就意味着一个团队可能由多个子团队组成。比如在Microsoft公司中有一个团队在做Office，这些团队中有的小团队在做Word，有的在做Excel，有的在做Access等等。在Elly的公司中Stancho是公司的老板，他的直接下属是Elly和Pesho。Elly是Kriss的经理。Pesho是Gosho 和Tosho的经理。根据上述的规则，Stancho, Elly, Pesho, Kriss, Gosho和Tosho形成一个团队；Elly和Kriss也形成一个团队；Pesho, Gosho和Tosho形成另一个团队。

Elly他们的公司刚刚搬到一个新的狭长的办公室。那里的空间只能放下一排电脑桌。老板Stancho早已占据了最左边的位置，并且他希望他的属下以下列方式分配座位：

    a）程序员的直接上司必须坐在这个程序员的左边；

    b）一个团队所有人的座位的编号必须是连续的

如果我们考虑Elly公司的情况，那么一个可能的坐法就是：Stancho, Pesho, Gosho, Tosho, Elly, Kriss。

Elly希望能给老板Stancho一个好印象，因此她希望你能帮助她找出一个安排员工的方案。当然她会告诉你公司的雇佣关系。

输入
输入文件company.in，共N行。
第一行一个正整数N，表示公司的员工总数（包括老板）。所有员工将会以1，2，…，N编号。当然老板的编号始终是1，并且他没有直接上司。
接下来N-1行每行两个整数W1和W2，表示W1是W2的直接上司。
输出
输出文件company.out，仅一行。
这一行包含N个共空格隔开的整数（范围为1~N），表示安排方案。如果存在多种安排方案，那么输出字典序最小的一组。安排A的字典序小于安排B当且仅当A和B序列中出现的第一个（最左边）不同的数时，A中的数比较小。比如序列{1, 3, 4, 6, 5, 2, 7}字典序小于序列{1, 3, 5, 2, 4, 7, 6}。

提示
【样例1解释】
你可以这样认为：Stancho标号为1, Elly标号为2, Kriss标号为5, Pesho标号为4, Gosho标号为3, Tosho标号为6。

【数据规模】
1 ≤ N ≤ 200,000
70%的数据满足1 ≤ N ≤ 10,000
85%的数据满足不超过200个程序员有相同的直接上司
