1047:Magic Tree

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
1000ms
内存限制:
65536kB
描述

Modular在他的生日派对上收到了一份特别的礼物——N个结点！这N个结点上分别标上了1..N这N个数字，并且每两个结点上标的数组都不一样。送这份礼物的Sths告诉Modular，这N个结点可以拼成一棵传说中的魔法树！魔法树有着非常强大的力量，能够帮助Modular实现很多很多的愿望，但是Sths却没有告诉Modular如何拼起这棵魔法树，于是Modular只有前往图书馆查找资料，经过漫长的努力，Modular找到了如下资料：

魔法树是一棵二叉搜索树，在搭建它的过程中，Modular需要从N个结点中选出一个作为魔法树的根，随后按一定的顺序将剩下的N-1个结点依次加入到魔法树中（即选取原树中一个节点作为这个节点的父亲节点），同时需要保证每一次加入结点后，魔法树都仍然保持着二叉搜索树的性质。(二叉搜索树的信息可见下方提示）

理论上无论以怎么样的顺序加入结点都可以顺利的搭建起魔法树，但是不同的搭建方式会导致魔法树实验愿望的能力产生变化，魔法树实现愿望的方式是这样的：

魔法树能实现的愿望有N种，第i种愿望需要魔法树中标号为i的结点来实现，在实现这种愿望的时候，魔法树会从根出发，利用身为二叉搜索树的性质来找到标号为i的结点，魔法能量会在这个过程中聚集，聚集的魔法能量与经过的结点数成正比，聚集的魔法能量越多，愿望就会实现的越好。

Modular列出了M个愿望的清单，希望能找到一种搭建魔法树的方式，使得实现这M个愿望所聚集的魔法能量的总量最多，但Modular对这方面一无所知，于是他找到了参加POJ挑战赛的你，你能帮助他么？

输入
每个文件包含多组数据
对于每一组数据，第一行有两个正整数N、M，意义如题目描述。
第二行有M个正整数，表示Modular的M个愿望的种类（保证两两不同）。
其中1 <= M <= N <= 1000
输出
对于每一组数据，输出一行，表示最多可以聚集的魔法能量。
样例输入
3 2
1 3 
3 2
3 2 
3 3
3 2 1
样例输出
4
5
6
提示
二叉搜索树是一种点上有权值的二叉树，其满足性质如下：对于任意节点Pi,其左子树中的所有点的权值都比它的权值小，其右子树中的所有点的权值都比它的大。
