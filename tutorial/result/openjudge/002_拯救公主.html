002:拯救公主

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
1000ms
内存限制:
65536kB
描述

lyc是一位超级英雄，他非常擅长营救美女。有一天，lyc不小心掉入了一个N*M的迷宫里，迷宫周围是无法逾越的高墙。迷宫内有许多稀奇古怪的机关，包括障碍物’#’、跳跃点’E’、以及公主。lyc可以按四个方向行走，每次行走花费1秒。lyc身上还带了V个可电池，每个电池可以为lyc提供1点能量。lyc发现跳跃点可以进行跳跃，设当前带有P点能量，那么到跳跃点时，可以不跳，也可以选择继续向前跳P个格子（跳跃方向必须和上一步方向一致）直到被高墙阻挡（障碍物无法阻挡），但是，如果lyc落到障碍物上则会死亡。lyc是个好公民，他不会随地丢垃圾，但是他非常想救公主，因此他任意时刻任意地点都可以扔电池，但是由于神秘的东方力量，扔的电池会消失。由于lyc是运动健将，因此lyc跳跃、扔电池都不耗费时间。lyc起始在‘S’，他想救在’T’的公主，求lyc的最短救援时间。


输入
输入有多组，每组第一行为三个整数N,M,V
（ 0 < N, M <= 100, 0 <=V <=10 )
下面是N行，每行M个字符，表示迷宫。
输出
对于每组输入，输出lyc的最短救援时间，无解则输出-1。
样例输入
5 4 3
T#..
##..
....
E.ES
....
1 11 3
SE..E..E#T.
2 3 0
S#T
...
2 2 0
S#
#T
0 0 0
样例输出
Case #1: 3
Case #2: 1
Case #3: 4
Case #4: -1
提示
Case 1:公主在(0,0)点，lyc在(3,3)先向左走，耗费一秒，在(3,2)的跳跃点跳跃，可以向左跳跃3格，但是由于高墙阻挡，lyc只能落到(3,0)的跳跃点，此时lyc选择不跳跃。为了向上跳跃，lyc走到(4,0)，耗费1秒，再走到(3,0)的跳跃点，耗费1秒，此时lyc选择跳跃，向前跳跃到(0,0)，拯救公主，总耗费3秒。
Case 2:公主在(0,9)点，lyc在(0,0)点先向右走，耗费1秒，此时lyc有3个电池，可以在(0,1)的跳跃点向右跳到(0,4)的跳跃点，再向右跳到(0,7)的跳跃点，跳跃中，lyc丢掉1个电池，因此在(0,7)的跳跃点，lyc有2个电池，可以向右跳到(0,9)的公主处，整个过程花费1秒。
