1068:GREEN Mountain Walking

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
1000ms
内存限制:
100000kB
描述
Farmer John和牛Bessie享受了最active的假期。他们花了一整天在山上漫步，最后疲惫地
回到他们假日的小屋。

    由于爬山需要很多体力，而且他们已经非常疲惫，所以他们想要穿过一条最高点和最低点高
差最小的路径回到他们的小屋，尽管这个样的路径或许很长。帮助FJ找到这样一条易于攀登的路
线！


    整座山的地图用一个N x N (2 <= N <= 100)的方阵给出。方阵中填充了一些描述海拔的整数
(0 <= 任意高度 <= 110)。 FJ和Bessie现在在地图的左上角（第一行，第一列）而小木屋在地图
的右下角（第N行，第N列）。他们可以向上走，向下走，向右走，向左走，但是不能穿越对角线。


输入
* 第一行： 一个单独的整数, N

* 接下来2..N+1行：
每一行包含N个整数，用来描述每一个方格的海拔。第二行包含地图上第一行（最上面一行）
的格子，第三行是地图上第二行的格子，依此类推。每一行的第一个数字表示地图中的第一
列，依此类推。
输出
* 一行：一个整数，表示路径上最高点和最低点的高度差。
样例输入
5
1 1 3 6 8
1 2 2 5 5
4 4 0 3 3
8 0 2 3 4
4 3 0 2 1
样例输出
2
提示
输出细节:

xx... 可以从这里看到，最低点是0，最高点是2，所以最大高度差是2。
.xx..
..x..
..x..
..xxx
