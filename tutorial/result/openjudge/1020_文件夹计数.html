1020:文件夹计数

  • 查看
  • 提交
  • 统计
  • 提问
总时间限制:
10000ms
单个测试点时间限制:
1000ms
内存限制:
256000kB
描述
CodeRush给他的笔记本电脑换上了Linux uBuntu操作系统，在Linux系统中，文件都存在文件夹里，系统已有一个根文件夹，在根文件夹里有不同名字的子文件夹，子文件夹中还有更多的文件夹……
可以用路径来描述文件夹，路径为一个包含若干部分的字符串，之间用’/’分隔。每部分均为一个文件夹的名称，且表示这个文件夹的父文件夹为前一部分描述的文件夹。
例如：/home/asez/coderush表示根目录下有一个名称为home的文件夹，这个home文件夹下有一个名称asez的文件夹，这个名称为coderush的文件夹下有。
每个路径的第1个字符总是’/’，且不会出现两个连续的’/’，最后一个字符不会是’/’。而所有文件夹仅包含数字和字母。
要在Linux中新建文件夹，可以用”mkdir”命令，当你输入一个路径来创建文件夹时，系统只会在该新文件夹的父目录存在时才会创建，例如，在没有任何目录的情况下，你想从头创建” /home/asez/coderush”和” /home/asez/addspop”两个文件夹，那么你需要以下四个命令：
mkdir /home
mkdir /home/ asez
mkdir /home/ asez / coderush
mkdir /home/ asez / addspop

CodeRush的笔记本电脑中现在已经有了n个文件夹，他还想新建m个文件夹，请你计算他需要使用多少次mkdir命令。
输入
输入文件,第一行一个整数T，表示有T组数据
每组数据的第1行为一个正整数n和m。
接下来n+m行，每行为一个描述路径的字符串，长度均不超过100。
根目录每台电脑都有，不用进行考虑。
数据保证一个目录不会在“已存在的目录”中出现两次，或者是在“待创建的目录中”出现两次。但是可能在“已存在的目录”中出现后，在“待创建的目录中”再次出现（见样例2）。
输出
输出应包含N行，每行1个正整数，第i行输出若要使第1个路径到第i个路径存在，最少需要新建多少个文件夹。
样例输入
3
0 2
/home/ asez / coderush 
/home/ asez / addspop
2 1
/chicken
/chicken/egg
/chicken
1 3
/a
/a/b
/a/c
/b/b
样例输出
4
0
4
提示
对于所有数据，0 ≤ N ≤ 100，1 ≤ M ≤ 100. 1 ≤ T ≤ 20.
对于部分数据，有N<=20；
对于部分数据，有对于所有路径最多存在两个’/’（包含第1个字符）。
