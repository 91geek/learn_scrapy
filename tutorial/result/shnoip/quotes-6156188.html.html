树

数据的逻辑结构分为线性结构和非线性结构。
线性结构即：各数据之间的逻辑关系可以用一个线性序列表示。
记录、文件、链表等都属于线性结构。
树、图属于非线性结构。

树为计算机应用中出现的具有层次关系和分支关系的数据提供了一种自然的表示。
树是一个或多个结点组成的有限集合T，有一个特定的结点称作根，其余结点可分为n>=0个不相交的集合T1，T2，…，Tn。每个集合又是一棵树，被称作根的子树。

树结构常用的术语有：
双亲、子女：结点的各子树的根称作该结点的子女，相应地该结点称作其子妇的双亲。
兄弟：具有相同双亲的结点互为兄弟。
结点的度：一个结点的子树的个数。
树叶：度为0的结点。
分支结点：度不为0的结点。
结点的层数：根结点的层数为1，其它任何结点的层数等于它的双亲结点的层数加1。
树的深度：树中结点的最大层数。

无序树不强调兄弟结点的先后次序，而二叉树是有序树。
二叉树是结点的有序集，可以为空集。
二叉树结点总数为：M=N0+N1+N2(N0、N1、N2分别表示度为0、1、2的结点数)。
所有结点的度不大于2的树并不一定是二叉树，必须附加条件：结点的子树区分左右，即使结点只有一棵子树，也要说明是左子树或右子树。

结论：
二叉树第i层最多有2的（i-1）次方个结点。
深度为k的二叉树，它的结点数目一定小于等于2的k次方-1。

特例：
空树和只有根结点的树也是二叉树。
如果一棵二叉树每一层上的结点个数都达到最大值，则该树称为满二叉树。
若一棵二叉树的结点中，只有最下两层的结点的度小于2，且最下层的结点都集中在最左边的若干位置，则该二叉树称为完全二叉树。满二叉树也是完全二叉树。

二叉树的llink-rlink法存储表示：
二叉树常用链表形式存储，每个结点的格式为：llink | info | rlink
info为结点本身的信息，llink和rlink为左子树和右子树根结点的指针。
完全二叉树的顺序存储：
基于完全二叉树的结构特点，通常采用顺序存储方式。

“ # ” 表示空结点，“ @ ” 表示输入结束。

二叉树的遍历是指按照一定的顺序访问二叉树的每个结点，并且每个结点只被访问一次。
先序遍历（NLR）：访问根结点—>先序遍历根结点的左子树—>先序遍历根结点的右子树。
中序遍历（LNR）：中序遍历根结点的左子树—>访问根结点—>中序遍历根结点的右子树。
后序遍历（LRN）：后序遍历根结点的左子树—>后序遍历根结点的右子树—>访问根结点。

算术表达式（a+b*（c-d））-e/f的二叉树结构：
先序遍历结果为：- + a * b - c d / e f ；即算术表达式的前缀表示法。
中序遍历结果为：a + b * c - d - e / f ；即算术表达式的中缀表示法。
后序遍历结果为：a b c d - * + e f / - ；即算术表达式的后缀表示法。
在先序遍历中根结点最先访问，后序遍历中根结点最后访问，而在中序遍历中，左右子树对称分布在根结点两侧。

先序、中序、后序遍历均属于深度优先遍历的范畴，我们也可以使用广度优先遍历树，即按层次次序遍历，首先访问层次为1的结点，然后依次访问层次为2的结点，直到访问完最后一层的所有结点。一般情况下，链式存储结构采用深度优先遍历，顺序存储结构采用广度优先遍历。
